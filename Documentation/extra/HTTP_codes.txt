### 1xx: Информационные (Informational)
Эти коды указывают, что сервер получил запрос и продолжает его обработку. Они являются временными и обычно не отображаются пользователю .

100
--> Значение: Continue (Продолжай)
--> Назначение: Сервер получил заголовки запроса, и клиент может отправлять тело запроса .
--> Обычное использование: Клиент отправляет заголовок `Expect: 100-continue`, чтобы проверить, примет ли сервер тело запроса (например, при загрузке большого файла), прежде чем отправлять его .
--> Реакция браузера: Браузер (клиент) отправляет оставшуюся часть запроса. Код не отображается пользователю.
--> Тело игнорируется: Да, тело ответа не предполагается.

101
--> Значение: Switching Protocols (Переключение протоколов)
--> Назначение: Сервер соглашается переключиться на протокол, указанный клиентом в заголовке `Upgrade` .
--> Обычное использование: Установка соединения WebSocket .
--> Реакция браузера: Браузер переключается на новый протокол (например, с HTTP на WebSocket).
--> Тело игнорируется: Да, тело ответа не предполагается.

102
--> Значение: Processing (Обработка)
--> Назначение: Сервер получил запрос и обрабатывает его, но ответ еще не готов (используется для предотвращения тайм-аута при длительных операциях) .
--> Обычное использование: Длительные запросы к серверу, например, сложные операции WebDAV .
--> Реакция браузера: Браузер ожидает финального ответа, соединение не разрывается.
--> Тело игнорируется: Да, это промежуточный ответ.

103
--> Значение: Early Hints (Ранние подсказки)
--> Назначение: Позволяет серверу отправить некоторые заголовки (чаще всего `Link` для предзагрузки ресурсов) до основного ответа .
--> Обычное использование: Сервер "подсказывает" браузеру, какие ресурсы (CSS, JS, изображения) нужно начать загружать, пока сервер генерирует основной HTML.
--> Реакция браузера: Браузер начинает предзагрузку указанных ресурсов, экономя время.
--> Тело игнорируется: Да, тело ответа не предполагается.

### 2xx: Успешные (Successful)
Эти коды информируют о том, что запрос клиента был успешно получен, понят и принят .

200
--> Значение: OK (Хорошо)
--> Назначение: Запрос выполнен успешно. Значение ответа зависит от метода запроса (GET, POST, HEAD и т.д.) .
--> Обычное использование: Самая частая ситуация — сервер отдал запрошенную веб-страницу или данные .
--> Реакция браузера: Браузер обрабатывает и отображает полученное содержимое.
--> Тело игнорируется: Нет, тело ответа содержит запрошенный ресурс или результат операции.

201
--> Значение: Created (Создано)
--> Назначение: Запрос выполнен успешно, и в результате был создан новый ресурс .
--> Обычное использование: Ответ на успешный POST-запрос при создании новой учетной записи, статьи или записи в базе данных. Новый URL часто указывается в заголовке `Location` .
--> Реакция браузера: Браузер получает подтверждение создания. В браузере это редко видно напрямую, чаще используется в API.
--> Тело игнорируется: Нет, тело может содержать описание или ссылку на созданный ресурс.

202
--> Значение: Accepted (Принято)
--> Назначение: Запрос принят в обработку, но она еще не завершена .
--> Обычное использование: Для пакетной обработки или длительных задач, выполняемых асинхронно (например, генерация отчета) .
--> Реакция браузера: Браузер понимает, что запрос принят, но не ждет результата немедленно.
--> Тело игнорируется: Нет, тело может содержать статус обработки или ссылку для проверки.

203
--> Значение: Non-Authoritative Information (Информация не авторитетна)
--> Назначение: Запрос выполнен успешно, но возвращаемые данные могут быть не от исходного сервера, а от промежуточного (например, прокси или кэша с модификациями) .
--> Обычное использование: Прокси-сервер, который изменяет содержимое (например, добавляет/удаляет рекламу), уведомляет клиента об этом .
--> Реакция браузера: Браузер обрабатывает ответ как успешный (аналогично 200).
--> Тело игнорируется: Нет, тело содержит (возможно, измененные) данные.

204
--> Значение: No Content (Нет содержимого)
--> Назначение: Запрос выполнен успешно, но сервер не отправляет никакого содержимого (тела ответа) .
--> Обычное использование: Сохранение данных на сервере (например, отправка формы через JavaScript), когда не нужно обновлять текущую страницу или показывать новое сообщение .
--> Реакция браузера: Браузер не меняет текущий документ/страницу .
--> Тело игнорируется: Да, по определению тело ответа отсутствует.

205
--> Значение: Reset Content (Сбросить содержимое)
--> Назначение: Сообщает клиенту необходимость сбросить (очистить) текущее представление документа, например, форму .
--> Обычное использование: После отправки формы через браузер (не через JS) сервер может ответить 205, чтобы очистить поля формы и подготовить её к новому вводу .
--> Реакция браузера: Браузер должен очистить текущую форму или перезагрузить фрейм.
--> Тело игнорируется: Да, тело ответа не должно содержать данных.

206
--> Значение: Partial Content (Частичное содержимое)
--> Назначение: Сервер успешно обработал частичный GET-запрос, содержащий заголовок `Range` .
--> Обычное использование: Возобновление прерванной загрузки, стриминг видео (пользователь перемотал, и браузер запрашивает только нужный фрагмент) .
--> Реакция браузера: Браузер принимает кусок данных и добавляет к уже имеющимся.
--> Тело игнорируется: Нет, тело содержит запрошенный диапазон данных.

207
--> Значение: Multi-Status (Многостатусный)
--> Назначение: Предоставляет статус для нескольких независимых операций в одном XML-теле ответа .
--> Обычное использование: В протоколе WebDAV, когда клиент выполняет несколько операций с файлами сразу.
--> Реакция браузера: Специфична для WebDAV-клиентов.
--> Тело игнорируется: Нет, тело (XML) содержит список статусов.

208
--> Значение: Already Reported (Уже сообщалось)
--> Назначение: Используется внутри ответа 207, чтобы избежать повторного перечисления одних и тех же ресурсов .
--> Обычное использование: Специфика протокола WebDAV.
--> Реакция браузера: Специфична для WebDAV-клиентов.
--> Тело игнорируется: Нет, является частью XML-структуры.

226
--> Значение: IM Used (IM использовано)
--> Назначение: Сервер успешно обработал GET-запрос и применил к ресурсу одну или несколько манипуляций (instance manipulations), результат чего возвращается в ответе .
--> Обычное использование: Используется с протоколом delta encoding в HTTP (редко).
--> Реакция браузера: Зависит от поддержки delta encoding.
--> Тело игнорируется: Нет, тело содержит результат манипуляции.

### 3xx: Перенаправления (Redirection)
Эти коды сообщают клиенту, что для выполнения запроса необходимо предпринять дополнительные действия, чаще всего перейти по другому URL .

300
--> Значение: Multiple Choices (Множество выборов)
--> Назначение: Запрошенный ресурс соответствует нескольким представлениям, и клиент должен выбрать одно из них (например, по языку или формату) .
--> Обычное использование: Сервер предлагает список возможных URL для одного ресурса, например, страницу на английском и на русском .
--> Реакция браузера: Браузер может автоматически выбрать предпочтительный вариант (если указан заголовок `Location`) или показать пользователю список для выбора.
--> Тело игнорируется: Нет, тело обычно содержит список ссылок для выбора.

301
--> Значение: Moved Permanently (Перемещено навсегда)
--> Назначение: Запрошенный ресурс был окончательно перемещен на новый URL, указанный в заголовке `Location` .
--> Обычное использование: Смена домена, переезд сайта на новую структуру URL. Передает вес ссылки (для SEO) на новый адрес .
--> Реакция браузера: Браузер автоматически выполняет GET-запрос на новый URL. При повторном обращении к старому адресу браузер сразу использует новый (кэширует редирект).
--> Тело игнорируется: Обычно да (браузер следует за редиректом), но тело может содержать ссылку на новый адрес.

302
--> Значение: Found (Найдено) [ранее "Moved Temporarily"]
--> Назначение: Запрошенный ресурс временно доступен по другому URL . В спецификации сказано, что метод запроса не должен меняться, но многие браузеры меняют его на GET.
--> Обычное использование: Временные редиректы, A/B тестирование, редиректы после успешного входа на главную .
--> Реакция браузера: Автоматически переходит по новому URL. Поведение с методами (POST -> GET) может отличаться .
--> Тело игнорируется: Обычно да, браузер следует за редиректом.

303
--> Значение: See Other (Смотреть другое)
--> Назначение: Ответ на запрос можно найти по другому URL, и его **обязательно** нужно получить с помощью метода GET .
--> Обычное использование: После отправки формы методом POST перенаправить пользователя на страницу "Спасибо за заявку", чтобы избежать повторной отправки при обновлении страницы .
--> Реакция браузера: Всегда выполняет GET-запрос к новому URL, независимо от исходного метода.
--> Тело игнорируется: Да, браузер следует за редиректом.

304
--> Значение: Not Modified (Не изменялось)
--> Назначение: Указывает, что ресурс не изменялся с момента последнего запроса. Клиент должен использовать свою кэшированную версию .
--> Обычное использование: Браузер запрашивает стиль style.css с заголовками `If-Modified-Since` или `If-None-Match`. Если файл не менялся, сервер отвечает 304, экономя трафик .
--> Реакция браузера: Браузер загружает ресурс из своего кэша.
--> Тело игнорируется: Да, тело ответа всегда должно быть пустым .

305
--> Значение: Use Proxy (Использовать прокси)
--> Назначение: **Устаревший.** Указывал, что ресурс должен быть доступен только через прокси-сервер . Сейчас не используется из-за угроз безопасности.
--> Обычное использование: Не используется.
--> Реакция браузера: Большинство браузеров просто игнорируют или обрабатывают как ошибку.
--> Тело игнорируется: Не применимо.

306
--> Значение: Switch Proxy (Переключить прокси)
--> Назначение: **Устаревший и зарезервированный.** Изначально предназначался для указания использовать другой прокси .
--> Обычное использование: Не используется.
--> Реакция браузера: Не обрабатывается.
--> Тело игнорируется: Не применимо.

307
--> Значение: Temporary Redirect (Временное перенаправление)
--> Назначение: Как и 302, указывает на временный редирект, но **строго запрещает** изменение HTTP-метода .
--> Обычное использование: Когда нужно временно перенаправить запрос, сохраняя исходные данные (например, POST-запрос с формой на временный обработчик) .
--> Реакция браузера: Повторяет запрос на новый URL с тем же методом и телом, что и исходный запрос.
--> Тело игнорируется: Нет, если исходный метод подразумевает тело (POST), оно будет отправлено снова.

308
--> Значение: Permanent Redirect (Постоянное перенаправление)
--> Назначение: Как и 301, указывает на постоянный редирект, но **строго запрещает** изменение HTTP-метода .
--> Обычное использование: При смене домена или структуры API, когда нужно гарантировать, что POST-запросы останутся POST-запросами и на новом месте .
--> Реакция браузера: Автоматически повторяет запрос на новый URL с тем же методом и телом. Кэшируется как постоянный редирект.
--> Тело игнорируется: Нет (в контексте повторной отправки POST-данных).

### 4xx: Ошибки клиента (Client Error)
Эти коды указывают, что ошибка произошла по вине клиента. Запрос содержит некорректные данные или не может быть выполнен по условиям доступа .

400
--> Значение: Bad Request (Плохой запрос)
--> Назначение: Сервер не может обработать запрос из-за синтаксической ошибки (например, кривая JSON структура, невалидный URL) .
--> Обычное использование: Отправка формы с неверно заполненными полями, которую сервер не может распарсить.
--> Реакция браузера: Показывает страницу с ошибкой 400.
--> Тело игнорируется: Нет, сервер может вернуть HTML с описанием ошибки.

401
--> Значение: Unauthorized (Не авторизован)
--> Назначение: Для доступа к ресурсу требуется аутентификация. Сервер должен указать метод аутентификации в заголовке `WWW-Authenticate` .
--> Обычное использование: Попытка зайти в админку без логина.
--> Реакция браузера: Показывает стандартное всплывающее окно для ввода логина и пароля (если используется базовая HTTP-аутентификация) или страницу ошибки .
--> Тело игнорируется: Нет, но браузер может перехватить ответ, чтобы показать диалог аутентификации.

402
--> Значение: Payment Required (Необходима оплата)
--> Назначение: Зарезервирован для будущего использования в системах с оплатой . В реальности почти не применяется, но иногда используется сторонними API для своих целей.
--> Обычное использование: Крайне редко.
--> Реакция браузера: Как обычная ошибка.
--> Тело игнорируется: Нет.

403
--> Значение: Forbidden (Запрещено)
--> Назначение: Сервер понял запрос, но отказывается его выполнять из-за прав доступа (недостаточно прав, IP в черном списке) . В отличие от 401, повторная попытка с теми же данными ничего не даст.
--> Обычное использование: Попытка открыть файл, к которому у вас нет доступа.
--> Реакция браузера: Показывает страницу с ошибкой 403.
--> Тело игнорируется: Нет, часто содержит причину отказа.

404
--> Значение: Not Found (Не найдено)
--> Назначение: Сервер не может найти запрошенный ресурс по данному URL .
--> Обычное использование: Переход по битой ссылке.
--> Реакция браузера: Показывает страницу с ошибкой 404.
--> Тело игнорируется: Нет, обычно содержит кастомную страницу 404.

405
--> Значение: Method Not Allowed (Метод не разрешен)
--> Назначение: Используемый HTTP-метод (например, PUT) не поддерживается для данного ресурса .
--> Обычное использование: Попытка отправить GET-запрос на API-эндпоинт, который принимает только POST.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет, в ответе обычно указывается заголовок `Allow` со списком разрешенных методов.

406
--> Значение: Not Acceptable (Неприемлемо)
--> Назначение: Сервер не может найти ответ, соответствующий списку допустимых значений, переданных клиентом в заголовках `Accept` (например, `Accept-Language`) .
--> Обычное использование: Запрос страницы на французском, когда сайт существует только на английском и русском.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет, обычно содержит список доступных вариантов.

407
--> Значение: Proxy Authentication Required (Необходима аутентификация прокси)
--> Назначение: Аналогично 401, но аутентификация требуется на прокси-сервере .
--> Обычное использование: Работа в корпоративной сети через прокси, требующий логин.
--> Реакция браузера: Показывает диалог для аутентификации на прокси.
--> Тело игнорируется: Нет.

408
--> Значение: Request Timeout (Тайм-аут запроса)
--> Назначение: Время ожидания запроса со стороны клиента истекло. Сервер закрывает соединение .
--> Обычное использование: Очень медленное интернет-соединение, из-за которого браузер не успел передать данные.
--> Реакция браузера: Показывает ошибку. Может автоматически повторить запрос позже .
--> Тело игнорируется: Нет.

409
--> Значение: Conflict (Конфликт)
--> Назначение: Запрос не может быть выполнен из-за конфликта с текущим состоянием ресурса .
--> Обычное использование: Редактирование документа, который был изменен другим пользователем (конфликт версий).
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет, обычно содержит информацию о конфликте.

410
--> Значение: Gone (Удалено)
--> Назначение: Ресурс был удален навсегда и больше не доступен. Серверу неизвестен новый адрес .
--> Обычное использование: Старая страница с акцией, которая закончилась, и товар убрали с сайта.
--> Реакция браузера: Показывает ошибку (аналогично 404).
--> Тело игнорируется: Нет.

411
--> Значение: Length Required (Необходима длина)
--> Назначение: Сервер отказывается принимать запрос без указания заголовка `Content-Length` .
--> Обычное использование: POST/PUT запросы, где серверу нужно точно знать размер тела.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

412
--> Значение: Precondition Failed (Предусловие ложно)
--> Назначение: Одно или несколько предусловий, указанных в заголовках запроса (например, `If-Match`), не выполнены .
--> Обычное использование: Загрузка файла, который с момента последней проверки был изменен на сервере (условие `If-Unmodified-Since` не сработало).
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

413
--> Значение: Content Too Large (Слишком большое содержимое) [ранее "Payload Too Large"]
--> Назначение: Тело запроса слишком большое, сервер отказывается его обрабатывать .
--> Обычное использование: Попытка загрузить фото размером 25 МБ на сайт, где лимит 5 МБ.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

414
--> Значение: URI Too Long (URI слишком длинный)
--> Назначение: URL запроса слишком длинный .
--> Обычное использование: Передача большого объема данных через GET (в URL), вместо POST. Сервер или промежуточный софт (например, Nginx) обрывает соединение.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

415
--> Значение: Unsupported Media Type (Неподдерживаемый тип данных)
--> Назначение: Формат данных, переданных в теле запроса, не поддерживается сервером .
--> Обычное использование: Отправка XML на API, которое принимает только JSON.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

416
--> Значение: Range Not Satisfiable (Диапазон не удовлетворим)
--> Назначение: Диапазон, запрошенный в заголовке `Range`, не может быть обработан. Например, запрос байт с позиции, превышающей размер файла .
--> Обычное использование: Сбой при возобновлении загрузки файла.
--> Реакция браузера: Показывает ошибку, возможно, попытается запросить файл заново целиком.
--> Тело игнорируется: Нет.

417
--> Значение: Expectation Failed (Ожидание не оправдалось)
--> Назначение: Сервер не может удовлетворить ожиданиям, указанным в заголовке `Expect` .
--> Обычное использование: Сервер не поддерживает `Expect: 100-continue`.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

418
--> Значение: I'm a teapot (Я - чайник)
--> Назначение: Код из первоапрельской шутки RFC 2324 (Hyper Text Coffee Pot Control Protocol). Сервер отказывается варить кофе, потому что он чайник .
--> Обычное использование: Используется для развлечения или как пасхалка.
--> Решаемый браузером: Может показать ошибку или специальную страницу.
--> Тело игнорируется: Нет.

421
--> Значение: Misdirected Request (Неверно направленный запрос)
--> Назначение: Запрос был направлен на сервер, который не может дать ответ (например, виртуальный хост на сервере не настроен для обработки этого домена) .
--> Обычное использование: Ошибка конфигурации сервера, особенно при использовании SNI.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

422
--> Значение: Unprocessable Entity (Необрабатываемый экземпляр)
--> Назначение: Сервер понял тип данных, но не может обработать инструкции (семантическая ошибка) .
--> Обычное использование: Отправка формы, где поле email содержит "abc", а не email. Сервер говорит: "Синтаксис верный, но данные валидацию не прошли".
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

423
--> Значение: Locked (Заблокировано)
--> Назначение: Текущий ресурс заблокирован .
--> Обычное использование: В протоколе WebDAV, файл редактируется другим пользователем.
--> Реакция браузера: Специфична для WebDAV-клиентов.
--> Тело игнорируется: Нет.

424
--> Значение: Failed Dependency (Ошибка зависимости)
--> Назначение: Выполнение запроса невозможно, так как оно зависит от другой операции, которая завершилась ошибкой .
--> Обычное использование: В WebDAV, если при выполнении пакета команд одна из них провалилась, для последующих может вернуться 424.
--> Реакция браузера: Специфична для WebDAV-клиентов.
--> Тело игнорируется: Нет.

425
--> Значение: Too Early (Слишком рано)
--> Назначение: Сервер не готов рисковать и обрабатывать запрос, который мог быть воспроизведен (replay attack) в рамках протокола TLS 1.3 .
--> Обычное использование: Связан с заголовком `Early-Data`.
--> Реакция браузера: Может повторить запрос без ранних данных.
--> Тело игнорируется: Нет.

426
--> Значение: Upgrade Required (Требуется обновление протокола)
--> Назначение: Сервер отказывается обрабатывать запрос с использованием текущего протокола и требует переключения на более новый, указанный в заголовке `Upgrade` .
--> Обычное использование: Сервер требует переход на HTTP/1.1 или HTTP/2.
--> Реакция браузера: Повторяет запрос с указанным протоколом.
--> Тело игнорируется: Нет.

428
--> Значение: Precondition Required (Необходимо предусловие)
--> Назначение: Сервер требует, чтобы запрос содержал условные заголовки (например, `If-Match`), чтобы избежать потери данных при конкурентном доступе .
--> Обычное использование: API для обновления ресурсов, предотвращающее конфликты версий.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

429
--> Значение: Too Many Requests (Слишком много запросов)
--> Назначение: Клиент превысил лимит количества запросов в единицу времени (rate limiting) .
--> Обычное использование: Защита API от DDoS-атак или чрезмерного использования.
--> Реакция браузера: Показывает ошибку. Может обработать заголовок `Retry-After` и повторить попытку позже.
--> Тело игнорируется: Нет.

431
--> Значение: Request Header Fields Too Large (Поля заголовка слишком большие)
--> Назначение: Общий размер заголовков запроса превышает допустимый лимит сервера .
--> Обычное использование: Слишком большой Cookie или Referrer.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

451
--> Значение: Unavailable for Legal Reasons (Недоступно по юридическим причинам)
--> Назначение: Доступ к ресурсу заблокирован по требованию госорганов или правообладателей .
--> Обычное использование: Сайт или страница заблокированы на территории страны по решению суда.
--> Реакция браузера: Показывает ошибку (иногда с пометкой о блокировке).
--> Тело игнорируется: Нет, обычно содержит объяснение причины блокировки.

### 5xx: Ошибки сервера (Server Error)
Эти коды сигнализируют о том, что сервер не смог выполнить корректный запрос из-за внутренней ошибки .

500
--> Значение: Internal Server Error (Внутренняя ошибка сервера)
--> Назначение: Универсальный код ошибки, когда сервер столкнулся с непредвиденным условием .
--> Обычное использование: Ошибка в коде PHP/Python, нехватка памяти на сервере, краш базы данных.
--> Реакция браузера: Показывает страницу с ошибкой 500.
--> Тело игнорируется: Нет, часто содержит технические детали (в режиме отладки) или общую страницу ошибки.

501
--> Значение: Not Implemented (Не реализовано)
--> Назначение: Сервер не поддерживает функционал, необходимый для обработки запроса .
--> Обычное использование: Использование HTTP-метода, который сервер вообще не поддерживает.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

502
--> Значение: Bad Gateway (Плохой шлюз)
--> Назначение: Сервер, выступая в роли шлюза или прокси, получил некорректный ответ от вышестоящего сервера .
--> Обычное использование: Прокси-сервер (например, Nginx) не смог получить данные от бэкенда (например, Apache, PHP-FPM), так как бэкенд "упал".
--> Реакция браузера: Показывает ошибку 502.
--> Тело игнорируется: Нет.

503
--> Значение: Service Unavailable (Сервис недоступен)
--> Назначение: Сервер временно не может обрабатывать запросы (перегрузка, технические работы) .
--> Обычное использование: Сервер "лежит" под высокой нагрузкой или выключен на техобслуживание.
--> Реакция браузера: Показывает ошибку. Может повторить запрос позже, если указан заголовок `Retry-After` .
--> Тело игнорируется: Нет.

504
--> Значение: Gateway Timeout (Тайм-аут шлюза)
--> Назначение: Сервер-шлюз не дождался ответа от вышестоящего сервера за отведенное время .
--> Обычное использование: Бэкенд-сервер (где крутится PHP) слишком долго обрабатывал скрипт, и веб-сервер (Nginx) прервал соединение по таймауту.
--> Реакция браузера: Показывает ошибку 504.
--> Тело игнорируется: Нет.

505
--> Значение: HTTP Version Not Supported (Версия HTTP не поддерживается)
--> Назначение: Сервер не поддерживает версию HTTP-протокола, использованную в запросе .
--> Обычное использование: Запрос HTTP/3 к серверу, настроенному только на HTTP/1.1 и HTTP/2.
--> Реакция браузера: Показывает ошибку. Может попробовать повторить запрос с более старой версией протокола.
--> Тело игнорируется: Нет.

506
--> Значение: Variant Also Negotiates (Вариант тоже участвует в согласовании)
--> Назначение: Ошибка конфигурации сервера при прозрачном согласовании содержимого (Transparent Content Negotiation). Выбранный ресурс указывает сам на себя в качестве ссылки .
--> Обычное использование: Редкая ошибка.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

507
--> Значение: Insufficient Storage (Недостаточно места)
--> Назначение: На сервере недостаточно места для выполнения запроса .
--> Обычное использование: В WebDAV при попытке загрузить файл на заполненный диск.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

508
--> Значение: Loop Detected (Обнаружен цикл)
--> Назначение: Сервер обнаружил бесконечный цикл при обработке запроса .
--> Обычное использование: Рекурсивный вызов, ошибка в настройках redirect-правил.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

510
--> Значение: Not Extended (Не расширено)
--> Назначение: Для выполнения запроса требуется дальнейшее расширение протокола, которое сервер не поддерживает .
--> Обычное использование: Связан с расширениями HTTP.
--> Реакция браузера: Показывает ошибку.
--> Тело игнорируется: Нет.

511
--> Значение: Network Authentication Required (Требуется аутентификация в сети)
--> Назначение: Клиент должен пройти аутентификацию для получения доступа к сети (например, в Wi-Fi hotspots с Captive Portal) .
--> Обычное использование: Подключение к публичной Wi-Fi сети в кафе, которая перехватывает запрос и перенаправляет на страницу авторизации.
--> Реакция браузера: Браузер перехватывает этот ответ и показывает окно для входа в сеть или открывает страницу авторизации.
